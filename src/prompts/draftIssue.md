# Draft GitHub Issue

Convert the following analysis result into a well-structured GitHub issue.

## Input
```json
{{analysis_json}}
```

## Requirements
- Title should be concise and actionable
- Include Summary, Impact, Evidence, Recommended Actions, and Done Criteria sections
- Use markdown formatting
- Include representative trace IDs for investigation
- Done criteria must be measurable
- Keep the tone technical and objective

## Output
Return the complete issue body in markdown format.
